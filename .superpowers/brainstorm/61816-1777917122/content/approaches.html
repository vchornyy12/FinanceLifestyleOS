3 Approaches to Multi-Format Receipt Parsing

Each approach changes how the API route sends content to Claude

A

Claude-Native Multi-Modal Recommended

Use Claude's built-in capabilities for every type — no new server-side libraries.

🖼️
JPEG/PNG/WebP
Vision API
(unchanged)
📄
PDF (multi-page)
Claude Document API
type: 'document'
📝
TXT / CSV
Text message

Pros

  • No new npm packages
  • Multi-page PDF natively
  • Simplest code path

Cons

  • PDF up to ~32 MB limit
  • Slight extra API latency for large PDFs
B

Server-Side PDF Text Extraction

Extract text from PDFs using pdfjs-dist on the server, then send all types as text.

🖼️
JPEG/PNG/WebP

(unchanged)

Extract text → Text message via pdfjs-dist

TXT / CSV
Text message
(direct read)

Pros

  • Works for any text-based PDF
  • Smaller Claude input for text PDFs

Cons

  • Adds pdfjs-dist (~2 MB bundle)
  • Fails for scanned/image-only PDFs
  • Loses visual layout info
