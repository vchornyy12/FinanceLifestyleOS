Design: API Route & Prompt Changes

Section 2 of 4 — route.ts and parseReceiptPrompt.ts

route.ts — what changes

1. Expand ALLOWED_MIME_TYPES
image/jpeg, image/png, image/webp
+ application/pdf
+ text/plain
+ text/csv
2. Size guard after file fetch
if pdf && buffer > 20 MB → 413 FILE_TOO_LARGE
if text && buffer > 1 MB  → 413 FILE_TOO_LARGE
3. Branch on MIME type before Claude call
image/*      → buildReceiptUserMessage(base64, mime)  ← existing
application/pdf → buildReceiptDocumentMessage(base64)     ← new
text/*       → buildReceiptTextMessage(utf8text)      ← new
4. MIME validation tightened
Currently, an unrecognised MIME silently falls back to image/jpeg. New behaviour: reject with UNSUPPORTED_FILE_TYPE (400) instead.

parseReceiptPrompt.ts — two new builders

buildReceiptDocumentMessage(pdfBase64)
content: [
 {
  type: 'document',
  source: {
   type: 'base64',
   media_type: 'application/pdf',
   data: pdfBase64
  }
 },
 { type: 'text', text: 'Parse this receipt...' }
]
buildReceiptTextMessage(text)
content: [
 {
  type: 'text',
  text: 'Parse this receipt and
       return JSON only.\n\n'
      + text
 }
]

System prompt — minor addition

Append one line to the existing rules:
- If input is CSV, the first row may be a header — skip it and parse data rows as items
Everything else in the prompt stays the same.
