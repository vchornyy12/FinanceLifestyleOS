Design: Architecture & Data Flow

Section 1 of 4 — The API route becomes a multi-modal router

Current flow (image only)

Client → upload JPEG/PNG/WebP → Supabase Storage
Client → POST storagePath → API route
API route → fetch file → base64 encode → Claude Vision → JSON

New flow (multi-format)

Client → upload any type → Supabase Storage
Client → POST storagePath → API route (multi-modal router)

API route → fetch file → detect MIME type
  ├── image/*      → base64 → Claude Vision      (unchanged)
  ├── application/pdf → base64 → Claude Document API  (new)
  └── text/plain|csv → UTF-8 decode → Claude Text msg  (new)

All paths → same JSON response → same validation → same review

What stays the same

✅ Storage path pattern
{userId}/{uuid}.ext — unchanged
✅ Auth & rate limiting
Bearer token + 20 req/hr — unchanged
✅ ParsedReceiptSchema
Same Zod validation on Claude output
✅ Review flow
Web review UI + mobile review screen — unchanged

File size guards (new)

🖼️
Images
No change
(mobile compresses to ~2 MB)
📄
PDF
Max 20 MB
(Claude supports 32 MB)
📝
TXT / CSV
Max 1 MB
(text receipts are tiny)
