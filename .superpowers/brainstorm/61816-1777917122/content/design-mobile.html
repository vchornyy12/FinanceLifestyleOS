New code — three small additions

1. receiptUpload.ts — uploadReceiptFile()
New function alongside uploadReceiptImage. Skips compression entirely. Picks the correct extension from the MIME type (pdf / txt / csv), uploads to receipts/{userId}/{uuid}.ext with the correct contentType.
2. useReceiptPipeline.ts — runFromFile(uri, mimeType)
New method on the existing hook. Skips compressImage, calls uploadReceiptFile instead of uploadReceiptImage, then calls the same parseReceipt → router.push review flow. The existing run() for camera/gallery is untouched.
3. index.tsx — handleFilePick()
New handler using expo-document-picker. Calls DocumentPicker.getDocumentAsync({ type: ['application/pdf','text/plain','text/csv'] }), then calls pipeline.runFromFile(uri, mimeType). Added to both the iOS ActionSheetIOS options and the Android Alert options.

New dependency

expo-document-picker
Already part of the Expo SDK ecosystem — no native build config changes needed. Added to apps/mobile/package.json.

What does NOT change on mobile

Camera capture flow, gallery flow, screenshot flow, review screen, save-receipt action — all untouched. The file upload path lands on exactly the same review screen as a camera photo.
